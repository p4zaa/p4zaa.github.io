LittleConfig Offerings


Modular Design: Specify main config files separately with sub-config files for better organization and clarity.

Intuitive Attribute Access: Access config attributes seamlessly with .attribute notation, making your code cleaner and more readable.
# Normal yaml config labels = config['labels']
# LittleConfig labels = config.labels

Dynamic Overrides: Override specific attributes on-the-fly without altering the main configuration, enhancing flexibility.
# Overrides overrides = {'mode': 'evaluate'}

Simplicity & Usability: Designed to be simple and easy to use, enabling quick adoption and integration.

Config-Oriented Approach

Advantage Description
Modularity Enable modular configuration management, simplifying updates and maintenance across different project components.
Reusability Reuse configurations across various projects, reducing redundancy and saving time.
Consistency Ensure consistent setups across different environments, minimizing the risk of configuration errors.
Separation of Concerns Keep configuration logic separate from application logic for cleaner, more maintainable code.
Ease of Maintenance Update configurations with ease by modifying config files rather than code.
Scalability Effortlessly scale your applications with different configurations for development, testing, and production environments.

Getting Started

Using LittleConfig is a breeze. Here's a quick example to get you started:

little_config = LittleConfig( initial_path = 'path/to/initial_yaml_path', config_path = 'path/to/config_path', config_name = 'sentiment-prediction-model1', overrides = {'mode': 'evaluate'})
config = little_config.config
